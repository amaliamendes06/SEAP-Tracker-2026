SEAP 2026 · First-time setup
Connect your database
This tracker needs a free Supabase database so all partners share the same data in real time. This takes about 10 minutes and is free. Follow the steps below.
01
Create a free Supabase account
Go to supabase.com → "Start your project" → sign up with your CDC Google account or email. Create a new project, give it any name (e.g. "seap-tracker"), choose a region close to Somalia (Europe West is fine), set a database password.
02
Create the activity_logs table
In your Supabase project, click "SQL Editor" in the left sidebar, paste the SQL below, and click Run.
create table activity_logs ( id uuid default gen_random_uuid() primary key, activity_id integer, activity_label text, indicator text, org text not null, status text not null, notes text, geography text, logged_at timestamptz default now() ); alter table activity_logs enable row level security; create policy "Anyone can read logs" on activity_logs for select using (true); create policy "Anyone can insert logs" on activity_logs for insert with check (true);
03
Get your project credentials
In Supabase go to Settings → API. You need two values: the "Project URL" and the "anon / public" key under "Project API keys". Paste them below.
Supabase Project URL
Supabase Anon Key
Your credentials are saved in this browser only (localStorage). Anyone using the tracker on a different device will need to enter them once too — or you can hardcode them in the HTML file before publishing to GitHub Pages.

The "anon" key is safe to share publicly — it only allows reading and inserting into the specific tables you created, controlled by the Row Level Security policies above.
